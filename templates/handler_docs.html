Donkey Handler Documentation

We want more than to just grab resources from the internet; we want specific BITS. Different resources need to be handled differently. As such, Donkey has different 'handlers' to deal with different resources.

XPATH

The most commonly used, as most of the resources we will want are HTML/XML documents. wraps Python's LXML package.

Specifically, this handler takes a document and turns it into lxml's etree object, then executes queries against it using the etree.xpath method

JMESPATH

For when you are working with JSON responses, and want to get a subsection from them, or want to construct crawl rules from them.JMESPath is a lush query language for JSON, and as fully featured as anyone could really ask for. It's astonishingly versatile, and you should consult the specification for instructions on how to use it.
The JMESPath handler merely wraps the search function of the JMESPath python package.
